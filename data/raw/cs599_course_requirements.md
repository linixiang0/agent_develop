# CS599 期末大作业要求知识库摘要

课程名称：企业级应用软件设计与开发。

课程代码：50120224001 / CS599。

学期：2025-2026 春季。

指导教师：戚欣。

## 项目方向

期末项目二选一：

1. 方向一：Agentic AI 原生开发。从零构建一个具备实际价值的 AI Agent 系统，运用 SDD 规格驱动开发方法论完成完整工程闭环。
2. 方向二：企业级应用软件的 Agent 改造。选择教务、电商、OA、ERP、物联网平台等企业系统，用 Agentic AI 技术进行智能化改造。

本项目选择方向一：Agentic AI 原生开发，项目名称为 EduRAG-Agent：面向研究生教务与课程资料的智能问答 Agent。

## 核心技术要素

项目至少涵盖以下技术要素中的 4 项：

- SDD 规格驱动开发。
- 工具使用、Function Calling 或 MCP 协议。
- 记忆机制，包括对话窗口、向量数据库或 MCP 外挂。
- 状态管理与多步骤推理，例如 LangGraph 或 ReAct。
- 多智能体协作。
- 可观测性与评估，例如 Tracing、Benchmark 或 LLMOps。

EduRAG-Agent 覆盖 SDD、工具调用、向量数据库记忆、Agent 状态管理、可观测性与评估。

## GitHub 仓库要求

每位学生必须在 GitHub 上创建课程大作业仓库，统一命名为 cs599-project。

如果是 Private Repository，必须添加 qxr777，也就是戚老师 GitHub 账号，为仓库 Collaborator。

如果是 Public Repository，确保包含开源协议文件，例如 MIT 或 Apache 2.0。

不得在代码中硬编码 API Key，必须使用环境变量。

## 报告要求

报告路径建议为 docs/CS599_大作业报告.pdf。

报告封面页必须包含课程名称、项目名称、方向、学号、姓名、专业、指导教师和提交日期。

提交日期为 2026 年 6 月 22 日。

报告 PDF 必须包含可用的导航窗格、书签、大纲或目录导航。

报告主要章节包括：

1. 选题背景与设计思想：问题定义、现有方案不足、项目价值、技术路线。
2. Specs 规格文档：Product Spec、Architecture Spec、API Spec。
3. 系统架构与设计：核心架构图、Agent 交互流程、数据流设计，强调图文并茂。
4. 关键实现与代码展示：Agent 核心循环、工具定义、配置文件、AI IDE 使用截图。
5. 测试与评估：功能测试、Agent 行为评估、Benchmark、Demo 截图或录屏。
6. 系统升级与扩展：可扩展架构、下一阶段计划、AI 能力演进路径。
7. 课程总结：个人收获、工程思维转变、对课程的建议。

## 评分标准

评分项包括：

- 选题与设计思想，20 分。
- Specs 规格设计，20 分。
- 系统架构与设计，15 分。
- 关键实现与代码，15 分。
- 测试与评估，10 分。
- 升级扩展设想，10 分。
- 课程总结，10 分。

加分项包括：

- 融合 MCP 协议或 Agentic RAG 等前沿技术，最高加 3 分。
- 云服务器部署并提供可访问 URL，最高加 3 分。
- 课堂分享展示，最高加 2 分。
- 达到生产级水平，包括错误处理、安全防护、可观测性，最高加 3 分。
