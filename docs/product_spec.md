# Product Spec：EduRAG-Agent

## 1. 产品定位

EduRAG-Agent 是一个面向研究生教务与课程资料的智能问答 Agent。系统将课程通知、培养方案、作业要求、FAQ 等资料构建为本地知识库，用户用自然语言提问后，Agent 自动检索证据、生成答案并附带来源。

## 2. 用户与场景

目标用户：

- 研究生学生：查询课程作业、提交节点、文档要求。
- 任课教师或助教：快速核对学生常见问题的依据。
- 项目评阅者：通过来源引用检查回答是否可信。

典型问题：

- “期末大作业最终提交截止时间是什么？”
- “报告 PDF 需要包含哪些章节？”
- “Private 仓库需要添加谁为 Collaborator？”

## 3. 痛点

课程资料分散在多个文件和通知中，人工查找效率低；普通聊天模型容易凭记忆回答，缺少课程依据；评阅时需要验证答案来源。

## 4. 产品目标

- P0：完成本地文档导入、切分、检索、问答闭环。
- P0：回答必须附带来源 chunk_id。
- P0：系统不硬编码 API Key，支持环境变量配置。
- P0：知识库区分课程要求、公开真实资料摘要和演示合成资料。
- P1：提供 Web 工作台，支持注册登录、多轮会话、知识库浏览和来源详情查看。
- P1：支持 OpenAI/DeepSeek 兼容 LLM。
- P1：提供评估脚本，输出来源命中率、关键词命中率、缺失项和耗时，并生成 Markdown 评估报告。
- P2：增加 MCP Server、云端部署、真实 embedding 检索和可视化 Trace。

## 5. 非目标

- 不做通用搜索引擎。
- 不回答知识库外的事实性问题。
- 不在代码仓库保存任何真实 API Key。

## 6. 成功指标

- 课程与教务综合测试集中关键词命中率不低于 0.8。
- 课程与教务综合测试集中来源命中率不低于 0.8。
- 典型问题平均响应时间小于 2 秒，本地模式下。
- 每次回答至少返回 1 个来源，除非知识库无相关资料。
- 评估集不少于 20 条，覆盖 CS599 评分要求、学籍、培养、论文、奖助、信息化和图书馆服务。
